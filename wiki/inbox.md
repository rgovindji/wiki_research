# Inbox — Async Capture

> **Purpose:** A scratch space for capturing notes, news links, ideas, and questions when away from the desktop. Drop anything here from your phone (via GitHub mobile web editor — tap the pencil icon on this file at github.com/rgovindji/wiki_research/blob/main/wiki/inbox.md). I'll process and clear this on the next desktop session.

## How to use from mobile

1. Open `https://github.com/rgovindji/wiki_research/blob/main/wiki/inbox.md` on your phone
2. Tap the **✏️ pencil icon** (top right of the file viewer)
3. Add a bullet under the relevant section below
4. Tap **"Commit changes"** at the bottom
5. Done — I'll process it on the next desktop session

**Or send the link to a Claude.ai mobile conversation** if you want to think it through first; copy the conversation summary here.

## Format

Each bullet should answer one of: **what is this**, **why it matters**, **what to do**.

- ✅ Good: `News: BABA Q4 earnings May 15 — cloud +28%; check read-through to GOOGL TPU thesis`
- ❌ Bad: `look at BABA`

Include URLs where available. Include dates if news-related.

## How I process this

When I see items here, I:

1. Read each item
2. Decide: action this session, defer (move to standing questions), or discard
3. Execute and move processed items to "✅ Processed" with a one-line note on what I did
4. Commit + push so the inbox stays clean

## Pro tips

- **Voice notes:** Record voice memos on your phone with thoughts. Transcribe via dictation → paste here. (Or skip transcription and paste to a Claude.ai conversation; share the conversation as a link here.)
- **Screenshots:** GitHub mobile web editor doesn't easily handle image upload, but you can paste a public image URL (X/Twitter posts, news sites) for me to fetch.
- **News links:** Just paste the URL with a 1-line description. I'll WebFetch the full content.
- **Speed:** Don't over-format. Bullet points are fine. I clean up before integrating to wiki pages.
